# 工作便签本 / DDL Assistant

一个纯前端的工作便签本与 DDL 辅助提醒工具。项目不依赖后端、不接入 Dify、不接入大模型 API、不使用数据库，所有数据都保存在浏览器 LocalStorage 中。

## 核心理念

- 先记录，再整理，最后才是任务。
- 左侧文本是事实来源，右侧 DDL 列表只是从文本中自动提取出的辅助视图。
- 勾选状态、完成时间等任务状态与左侧原文解耦，不会改写原始随记。

## 主要功能

- 本周随记按自然周分片保存。
- 长线项目与商务节点备忘长期保存。
- 支持从随记和长线备忘中统一提取 DDL。
- 支持冒号项目前缀批量继承。
- 支持日期域继承。
- 支持 `//` 后备注截断。
- 支持任务完成状态、完成时间记录。
- 支持 DDL 具体时间识别，例如 `10点`、`14:00`、`明天10点`、`6.20 14:30`。
- 支持今日、本周、未来、历史已完成分组。
- 支持本地 JSON 备份和恢复。
- 支持 PWA 安装和离线缓存。

## 轻量提醒限制

提醒能力是纯前端本地提醒，仅在应用打开期间运行。

- 只有存在具体时间的 DDL 才会触发提醒，例如 `6.20 14:30`、`明天10点`。
- 浏览器页面或已安装 PWA 必须处于打开状态。
- 浏览器完全关闭、系统休眠、设备关机、通知权限关闭时，不承诺仍能提醒。
- 到点提醒会优先使用 Service Worker 的 `showNotification`。
- 如果 Service Worker 通知不可用，会退回页面内提示。
- 本地数据仍保存在浏览器 LocalStorage 中，请定期使用“导出备份”保存 JSON 文件。

## 本地运行

可直接打开 `index.html` 使用基础功能。

如果需要 PWA 安装和通知能力，请通过本地服务器访问：

```powershell
python -m http.server 8000
```

然后打开：

```text
http://localhost:8000/
```

也可以双击项目中的 `start-app.cmd` 启动本地服务。

## 本地一键启动

Windows 用户可以双击 `start-app.cmd` 启动本地服务器并自动打开应用。

服务运行期间，代码修改后只需刷新浏览器；缓存未更新时可使用 `Ctrl+F5`。

也可以右键 `start-app.cmd`，为其创建桌面快捷方式。

## 文件结构

- `index.html`：页面结构。
- `style.css`：界面样式和窄屏布局。
- `app.js`：核心逻辑，包括文本解析、任务状态、本地提醒、备份恢复。
- `manifest.json`：PWA 应用配置。
- `sw.js`：Service Worker 离线缓存和通知点击处理。
