# 工作便签本 / DDL Assistant

一款面向办公室知识工作者的轻量工作便签本。

直接记录完整工作内容，系统自动提取日期、时间、项目归属和待跟进事项，生成今日、本周、未来与历史 DDL 视图。用户无需把一段工作安排拆成多条 Todo，也可以保留原始上下文。

- **在线体验：** https://yadingw.github.io/ddl-assistant/
- **当前版本：** V1.0
- **版本发布：** https://github.com/yadingW/ddl-assistant/releases

## 为什么做这个产品

日常工作安排常分散在沟通记录、会议记录和临时便签中。

传统 Todo 工具通常要求逐条创建任务、填写日期和维护状态；Excel 能管理节点，但需要先打开软件、找到对应工作簿，再手动维护。普通便签记录很快，却容易让日期节点沉入长文本。

DDL Assistant 尝试保留便签的低输入成本，同时补充自动日期提取和时间视图：

> 优先记录，其次整理，最后才是任务。

## 核心使用方式

在左侧直接写下或粘贴工作内容：

```text
项目A：

6.24 完成方案初稿
6.26 确认会议材料

> 同事A 6.25 确认反馈意见
// 这里可以保留背景说明，不进入任务标题
```

系统会在右侧生成对应提醒：

```text
[项目A] 完成方案初稿
[项目A] 确认会议材料
[项目A] 确认反馈意见
```

任务可以按今日、本周、未来和历史查看；点击任务正文可以定位回原始随记。

## 产品原则

- **连续记录优先：** 支持整段粘贴和持续编辑，不要求逐条创建任务。
- **原文保留上下文：** 左侧原文是事实来源，右侧任务是从原文派生的辅助视图。
- **状态独立保存：** 勾选状态和完成时间不会自动改写原文。
- **时间驱动推进：** 日期承担主要排序和提醒作用，项目上下文用于区分同名任务。
- **控制产品范围：** V1.0 暂不加入标签、优先级、项目看板和复杂多层任务结构。

## V1.0 主要功能

### 工作记录

- 本周随记按自然周保存；
- 长线事项备忘长期保存；
- 上周随记可查看和修改；
- 输入内容自动保存在浏览器本地。

### 日期与任务提取

- 支持完整日期、月日、相对日期和星期表达；
- 支持具体时间，例如 `10点`、`14:00`、`明天10点`、`6.20 14:30`；
- 支持日期域继承；
- 支持冒号项目上下文继承，并在任务标题前显示项目上下文；
- 支持 `>` 标记待跟进事项；
- 支持 `//` 排除整行备注或截断行内说明。

### 任务与历史

- 今日、本周、未来和历史已完成分组；
- 记录任务完成状态和完成时间；
- 当天完成的任务继续保留在今日视图；
- 更早完成的任务进入历史归档；
- 支持从任务定位到对应原文；
- 支持复制今日待跟进事项。

### 本地能力

- PWA 安装与独立窗口；
- Service Worker 离线缓存；
- 应用运行期间的到点提醒；
- JSON 数据导出、导入校验与失败回滚。

## 当前验证情况

V1.0 已完成发布前人工验收，并进入持续验证阶段。

当前已确认：

- 连续随记能够支持日常记录流程；
- PWA 独立入口降低了打开阻力；
- 常见日期、时间和项目表达可以覆盖典型场景；
- 桌面同类事项可集中在一个视图中查看；
- 临时记录可在后续整理时汇总到应用中。

当前正在验证：

- 连续便签与回车投递分别适合哪些用户；
- 待跟进功能是否具有持续价值；
- 使用者是否需要学习输入规则；
- 自然语言任务解析是否值得进入 AI POC。

## 技术实现概览

V1.0 采用纯前端实现：

```text
用户输入原文
    ↓
LocalStorage 自动保存
    ↓
本地规则解析日期、时间、项目和待跟进事项
    ↓
生成派生任务
    ↓
今日 / 本周 / 未来 / 历史视图
    ↓
独立保存完成状态与 completedAt
```

主要技术文件：

- `index.html`：页面结构；
- `style.css`：页面样式与窄屏布局；
- `app.js`：文本解析、任务状态、历史归档、本地提醒和备份恢复；
- `manifest.json`：PWA 配置；
- `sw.js`：离线缓存与通知点击处理。

V1.0 不依赖后端和数据库，也未接入大模型 API。用户数据保存在当前浏览器的 LocalStorage 中。

## 已知限制

- 数据不会自动跨浏览器或跨设备同步；
- 清理浏览器站点数据可能造成记录丢失，请定期导出 JSON 备份；
- 到点提醒仅在浏览器页面或已安装 PWA 运行期间检查；
- 浏览器关闭、系统休眠、设备关机或通知权限关闭时，无法保证提醒；
- 当前提供上周随记入口，更早历史周尚无完整检索和编辑界面；
- 数据增加后，启动阶段的缓存重建与集中渲染仍有优化空间；
- 复杂或模糊自然语言暂不由 AI 解析。

## 本地运行

直接打开 `index.html` 可以使用基础功能。

如需 PWA 安装和通知能力，请在项目目录启动本地服务器：

```powershell
python -m http.server 8000
```

然后访问：

```text
http://localhost:8000/
```

Windows 用户也可以双击：

```text
start-app.cmd
```

服务运行期间，修改代码后刷新页面即可；缓存未更新时可使用 `Ctrl+F5`。
